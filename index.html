06 ‚Äî White-Label Architecture
What Each Tenant Gets
Their own login credentials and isolated data
Business logo + primary brand colour applied throughout the UI
Custom T&C text shown on every sale
Their own payment gateway API keys (you never touch their money)
Custom invoice header with business name, address, VAT number
Subdomain option: theirbusiness.yourapp.com
Their own client + inventory database, private to them
What You (Super Admin) Get
Super admin dashboard to create / suspend / manage tenants
Subscription / billing layer per tenant (Stripe billing)
Usage analytics across all tenants
Feature flags ‚Äî enable/disable features per tenant plan
Global gateway defaults you pre-configure for clients
Support access (read-only tenant view) for troubleshooting
Onboarding flow to spin up a new tenant in < 5 minutes
Suggested Pricing Tiers (for your clients)
Starter: 1 user, 1 gateway, 50 transactions/mo, basic branding
Pro: 3 users, all gateways, unlimited transactions, full branding, PDF invoices
Business: 10 users, multi-staff roles, API access, custom domain, priority support
Enterprise: Unlimited, white-glove setup, SLA, custom features
Key Technical Decisions
Framework: React + Vite (PWA-ready, fast, mobile-first)
Backend: Firebase (Auth + Firestore + Storage + Functions)
PDF: jsPDF or React-PDF for client-side PDF generation
SMS: Africa's Talking (best for ZA/KE) or Twilio
Hosting: Firebase Hosting or Vercel (CDN edge, global)
Webhooks: Firebase Functions to receive gateway callbacks
